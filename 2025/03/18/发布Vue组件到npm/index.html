涉及的概念

node

Node.js 是一个基于 Chrome V8 引擎的 JavaScript 运行环境， 突破了浏览器限制 ，使得 JavaScript 能够操作本地文件、创建 HTTP 服务等。

开发中的具体作用 ：

  • 运行 vite 或 webpack 等构建工具
  • 执行 npm run dev 等开发脚本
  • 托管本地开发服务器（如通过 vite 启动的 localhost:3000）

npm

核心定位 ：
npm 是 包管理工具 + 软件注册表 的复合体：

  1. 注册表（Registry） ：存储了数百万个开源包（如 vue、lodash），类似“应用商店”
  2. 命令行工具（CLI） ：提供 install、publish 等命令管理依赖

关键功能补充 ：

  • 版本控制（^1.0.0 表示兼容版本）
  • 依赖关系解析（自动安装子依赖）
  • 脚本执行（package.json 中的 scripts 字段）

比喻理解 ：

  • npm ≈ Android应用商店（注册表） + 手机上的应用管理器（CLI）

vite

Vite 官方中文文档

  • 开发时 ：是浏览器与开发者之间的“ 智能桥梁 ”，利用现代浏览器能力跳过打包。
  • 生产时 ：是“ 优化器 ”，沿用传统构建工具的优势生成高性能代码。
  • 框架支持 ：是“ 乐高底座 ”，通过插件自由拼装能力，适应任何技术栈。

1. 开发阶段：轻量级服务器（非传统打包器）

  • 角色 ：按需编译的 ESM 服务器。
  • 工作原理 ：
  • 直接利用浏览器原生 ESM 能力，将源码文件（如 .vue、.jsx）通过插件实时编译为浏览器可识别的 ESM 模块。
  • 依赖项（如 node_modules 中的库）通过 esbuild 预构建为 ESM 格式并缓存。
  • 结果 ：无需打包整个应用，实现秒级启动和即时热更新。

2. 生产阶段：构建工具（基于 Rollup）

  • 角色 ：高性能打包器。
  • 工作原理 ：
  • 使用 Rollup 对代码进行打包、Tree-Shaking、代码分割等优化。
  • 复用开发阶段的插件体系（如 Vue/React 插件），确保构建行为一致。
  • 结果 ：生成高度优化的静态文件（如 index.html + 分块 JS/CSS）。

3. 框架无关性：插件化扩展

  • 核心定位 ：Vite 本身是一个 构建工具链框架 ，不绑定任何特定框架。
  • 对框架的支持 ：
  • 官方插件 ：如 @vitejs/plugin-vue（支持 Vue 单文件组件）、@vitejs/plugin-react（支持 React JSX）。
  • 社区插件 ：支持 Svelte、SolidJS 等新兴框架。
  • 插件的作用 ：
  • 解析框架特定语法（如 .vue 文件中的 <template>、<script>、<style>）。
  • 集成框架生态工具（如 Vue Router、React Refresh）。

为什么说它是“基础框架”？

  1. 核心功能与框架解耦 ：
  • Vite 的核心仅处理 开发服务器启动 、 模块编译管道 、构建流程 等基础能力。
  • 框架特性（如 Vue 的 SFC、React 的 Fast Refresh）通过插件注入到核心流程中。
  1. 插件系统的标准化 ：

  • 插件通过 Vite 的 Plugin API 介入编译过程。

  • 例如，@vitejs/plugin-vue 的工作流程：

    当浏览器请求 .vue 文件时，Vite 调用该插件将文件拆解为 JS/CSS，再分别处理。

  1. 生态兼容性 ：
  • 开发者可自由组合插件（如同时使用 @vitejs/plugin-vue + vite-plugin-pwa），无需被框架限制。
  • 社区插件覆盖场景广泛：Markdown 编译、SVG 图标处理、SSR 支持等。

示例

1、创建框架

参考资料：

vite

vue3

引入 vite vue 和相关插件

1.执行 npm create vue 命令，创建 vue 项目。
2.输入项目名称 yshboard
3.选择 TypeScript 功能
4.项目初始化完成，可执行 npm install 安装依赖，npm run dev 启动项目。
5.调整项目框架,清除默认的 HelloWorld.vue 文件，App.vue清除默认代码。

├── src
│ ├── components //组件文件夹
│ │ ├── Line // 一个组件
│ │ │ ├── index.ts //组件的导出文件
│ │ │ ├── Line.vue //组件的实现文件
│ │ ├── index.ts //所有组件的导出文件
│ ├── theme //echarts的主题文件夹
│ ├── App.vue
│ ├── main.ts
├── package.json // 项目的依赖 和 npm发布的配置文件
├── README.md // 项目的说明文件
├── tsconfig.app.json
├── tsconfig.json // typescript的配置文件
├── tsconfig.node.json
├── vite.config.ts // vite的配置文件

vite

  • vite.config.ts

5、测试打包

  • 配置 package.json
  • 在项目根路径下执行 npm link ,然后再测试项目中使用 npm link yshboard

6、发布到 npm

  • 先将npm的镜像地址改为npm官网
  • 执行 npm login 登录
  • 执行 npm publish --access=public发布到 npm上
  • 在测试项目中从npm按照进行测试
